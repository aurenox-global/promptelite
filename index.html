§ 01

Fundamentos

Los 6 Principios que lo Cambian Todo. Antes de escribir cualquier prompt, estos principios deben estar integrados en tu forma de pensar.

Contexto: Rol + Tarea + Formato

Siempre define quién eres, qué quieres y cómo lo quieres. Sin estos 3 elementos, la IA adivina. Cada token de contexto que ahorras, lo pagas en iteraciones.

+100% control de output

Densidad: Máxima Información

Elimina cortesías, relleno y repeticiones. "Por favor, podrías..." cuesta tokens. "Dame..." es suficiente. La IA no necesita amabilidad.

-70% tokens basura

Límites: Di qué NO quieres

Los límites negativos son igual de importantes que las instrucciones. "Sin introducción, sin conclusión, sin disculpas" evita relleno automático.

-30% tokens de corrección

Especificidad: Números Concretos

"Breve" significa 50 palabras para ti y 400 para la IA. "Dame 3 puntos de máximo 20 palabras cada uno" no deja margen de interpretación.

+85% precisión

Cadena: Divide Tareas Complejas

Un prompt largo = resultado mediocre. Usa chain-of-thought: primero el esquema, luego cada parte. La IA trabaja mejor con instrucciones atómicas.

+calidad ×5

Feedback Loop: Evalúa y Refina

Si el primer output tiene el 80%, pide el 20% faltante directamente. "Añade manejo de errores al bloque anterior" es más eficiente que reescribir.

-3 ciclos de corrección

# Escribe función fibonacci(n) recursiva con memoización 
# Docstring. Type hints. Sin explicación. 
def fibonacci(n: int, memo: dict = None) -> int:
     """Retorna el n-ésimo número de Fibonacci."""
     if memo is None: memo = {}
     if n in memo: return memo[n]
     if n <= 2: return 1 
    memo[n] = fibonacci(n-1, memo) + fibonacci(n-2, memo)
     return memo[n]

SQL Complejo — Una Línea 
-- TOP 5 clientes con mayor gasto en 2024
-- Tablas: clientes(id,nombre), pedidos(id,cliente_id,total,fecha)
-- Incluye CTE. Solo query.
WITH cliente_gasto AS (
   SELECT c.id, c.nombre, SUM(p.total) AS total_gastado
   FROM clientes c
   JOIN pedidos p ON c.id = p.cliente_id
   WHERE p.fecha BETWEEN '2024-01-01' AND '2024-12-31' 
   GROUP BY c.id, c.nombre
)
SELECT * FROM cliente_gasto
ORDER BY total_gastado DESC LIMIT 5;
Video Processing Pipeline 
//  <task>Pipeline detección facial en stream</task>
//  <constraints>Face-api.js, webcam 720p, 15fps, WASM</constraints>
async function * faceDetectionPipeline(stream) {
   const detector = await faceDetection.createDetector(...)
   const track = stream.getVideoTracks()[0]
   const processor = new MediaStreamTrackProcessor(track)
   const reader = processor.readable.getReader()
  
   while (true) {
     const {done, value: frame} = await reader.read()
     if (done) break
     const faces = await detector.estimateFaces(frame)
     yield {timestamp: frame.timestamp, faces, buffer: frame}
    frame.close()  // Crítico: liberar memoria GPU
  }
}

§ 05

Prompts para Texto

El texto tiene más variables subjetivas que el código. Estos patrones los convierten en especificaciones objetivas.

❌ TEXTO AMBIGUO

"Escríbeme algo sobre los beneficios del trabajo remoto para publicar en LinkedIn. Que sea bueno."

⚠️ Sin voz, sin audiencia, sin límites.

✅ TEXTO CONTROLADO

"Post LinkedIn sobre trabajo remoto. Audiencia: CTOs y engineering managers. Tono: directo, sin motivational BS. Largo: 150-180 palabras."

✅ Output predecible. Voz, estructura y límites definidos.

Control de Voz y Tono

voice-control.txt 
Voz: [Técnica / Conversacional / Académica]
Persona: [Experto / Colega / Mentor]

Registros prohibidos:
- Sin frases de relleno ("en el dinámico mundo de...")
- Sin conclusiones moralizantes
- Sin uso de: "robusto", "sinergias", "holístico"

Fórmula de densidad:
Cada párrafo = 1 idea. Si hay 2 ideas = 2 párrafos.

Sector: e-commerce moda. Tono: cercano, no desesperado.

§ 08

Errores que Queman Tokens

Los errores más comunes y cuántos tokens desperdician por cada vez que los cometes.

"Actúa como..." sin contexto

Decir "actúa como un experto" sin definir qué expertise, en qué sector, con qué metodología = rol vacío que no filtra nada.

~40 tokens/iteración desperdiciados

Pedir "hazlo mejor" sin especificar

La IA no sabe qué es "mejor" para ti. Di exactamente: "reduce longitud 50%", "cambia tono a formal".

~150 tokens/ciclo desperdiciados

No limitar longitud de respuesta

Sin límite, la IA genera hasta el máximo permitido. Si necesitas 3 líneas y genera 20 párrafos = tokens quemados.

~800 tokens/exceso desperdiciados

Pegar contexto sin delimitar

Si pegas 2000 palabras de contexto sueltas, la IA no sabe dónde empieza tu instrucción. Usa siempre delimitadores.

~300 tokens de confusión

Usar "piensa paso a paso" sin necesidad

CoT quema 200-500 tokens extra. Úsalo SOLO para matemáticas o lógica compleja. Para texto = desperdicio.

~350 tokens innecesarios

Repetir contexto en cada prompt

No repitas en cada prompt el contexto completo del proyecto. Usa un "system prompt" o bloque de contexto fijo.

~200 tokens/prompt desperdiciados
