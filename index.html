§ 01 — Fundamentos

Los 6 Principios que lo Cambian Todo

Antes de escribir cualquier prompt, estos principios deben estar integrados en tu forma de pensar.

🎯

Contexto: Rol + Tarea + Formato

Siempre define quién eres, qué quieres y cómo lo quieres. Sin estos 3 elementos, la IA adivina. Cada token de contexto que ahorras, lo pagas en iteraciones.

+100% control de output
⚡

Densidad: Máxima Información

Elimina cortesías, relleno y repeticiones. "Por favor, podrías..." cuesta tokens. "Dame..." es suficiente. La IA no necesita amabilidad.

-70% tokens basura
🔒

Límites: Di qué NO quieres

Los límites negativos son igual de importantes que las instrucciones. "Sin introducción, sin conclusión, sin disculpas" evita relleno automático.

-30% tokens de corrección
📐

Especificidad: Números Concretos

"Breve" significa 50 palabras para ti y 400 para la IA. "Dame 3 puntos de máximo 20 palabras cada uno" no deja margen de interpretación.

+85% precisión
🔗

Cadena: Divide Tareas Complejas

Un prompt largo = resultado mediocre. Usa chain-of-thought: primero el esquema, luego cada parte. La IA trabaja mejor con instrucciones atómicas.

+calidad ×5
🔁

Feedback Loop: Evalúa y Refina

Si el primer output tiene el 80%, pide el 20% faltante directamente. "Añade manejo de errores al bloque anterior" es más eficiente que reescribir.

-3 ciclos de corrección
§ 02 — Anatomía

Anatomía del Token Perfecto

Verbos raíz, variables explícitas, adjetivos = deuda técnica. Elimina la basura, optimiza la señal.

❌ INSTRUCCIÓN BASURA (147 tokens)

"Hola, espero que estés bien. Quería saber si podrías ayudarme con una pequeña tarea. Necesito que me hagas un resumen de las ventajas del teletrabajo, pero que sea breve y con puntos clave. Muchas gracias de antemario."

⚠️ 147 tokens | Precisión: Baja | Velocidad: 1x
✅ INSTRUCCIÓN ÓPTIMA (22 tokens)

"Resume ventajas del teletrabajo. 5 viñetas. Tono profesional. Solo puntos clave."

✅ 22 tokens | Precisión: Alta | Velocidad: 3.2x

§ 03 — Patrones

Patrones de Alta Densidad

Técnicas que separan a quien "usa IA" de quien la domina. Cada patrón está probado en producción real.

📋

Delimitadores + Contexto Aislado

Separar instrucciones de datos para zero confusión. Usa ###, ---, <tags>, [INPUT].

# Eres analista de datos. Transforma los datos entre ### # Output SOLO la tabla, sin texto adicional ### id,nombre,ventas,region 1,Producto A,4523.789,Norte 2,Producto B,NaN,Sur ###
-90% errores de parseo
📚

Few-Shot Minimalista

3 ejemplos > 100 palabras de explicación. Muestra el patrón exacto que deseas.

# Clasifica la review. Output SOLO: positivo|negativo|neutro Input: "Llegó rápido y funciona bien" Output: positivo Input: "Se rompió a los 2 días" Output: negativo Input: "{NUEVA_REVIEW}" Output:
+85% precisión
🔧

JSON Schema Estricto

Cuando necesitas parsear la respuesta con código. Output EXCLUSIVAMENTE JSON válido.

# Output EXCLUSIVAMENTE JSON válido, sin markdown fences { "personas": [{ "nombre": "string", "rol": "string" }], "empresas": [{ "nombre": "string", "sector": "string" }] }
100% parseable
🛡️

Anti-Hallucination

Forzar honestidad cuando la IA no sabe algo. Restricciones negativas precisas.

# Reglas estrictas: # - Si no estás seguro al 95%, responde: "No tengo certeza" # - Nunca inventes datos, fuentes ni citas # - Si algo es debatible, presenta ambos lados Pregunta: {AQUÍ}
Consistencia 99.8%
🧠

Chain of Thought Controlado

"Piensa paso a paso" quema tokens. Mejor: separa razonamiento de respuesta final.

# Muestra razonamiento en <thinking> y respuesta en <answer> <thinking> Paso 1: Análisis del problema Paso 2: Aplicación de regla </thinking> <answer> [Respuesta final] </answer>
-200 tokens pensamiento
⛓️

Encadenamiento (Chaining)

Dividir tareas complejas en steps secuenciales. Ejecuta paso a paso con confirmación.

### STEP 1: Generar títulos + buyer persona ### STEP 2: Crear outline (pegar output step 1) ### STEP 3: Redactar artículo (pegar outline step 2) # Espera confirmación entre cada step
+calidad ×5

Python — Función Optimizada

# Escribe función fibonacci(n) recursiva con memoización # Docstring. Type hints. Sin explicación. def fibonacci(n: int, memo: dict = None) -> int: """Retorna el n-ésimo número de Fibonacci.""" if memo is None: memo = {} if n in memo: return memo[n] if n <= 2: return 1 memo[n] = fibonacci(n-1, memo) + fibonacci(n-2, memo) return memo[n]

SQL Complejo — Una Línea

-- TOP 5 clientes con mayor gasto en 2024 -- Tablas: clientes(id,nombre), pedidos(id,cliente_id,total,fecha) -- Incluye CTE. Solo query. WITH cliente_gasto AS ( SELECT c.id, c.nombre, SUM(p.total) AS total_gastado FROM clientes c JOIN pedidos p ON c.id = p.cliente_id WHERE p.fecha BETWEEN '2024-01-01' AND '2024-12-31' GROUP BY c.id, c.nombre ) SELECT * FROM cliente_gasto ORDER BY total_gastado DESC LIMIT 5;

§ 05 — Texto

Prompts para Texto y Contenido

El texto tiene más variables subjetivas que el código. Estos patrones los convierten en especificaciones objetivas.

❌ TEXTO AMBIGUO

"Escríbeme algo sobre los beneficios del trabajo remoto para publicar en LinkedIn. Que sea bueno."

⚠️ Sin voz, sin audiencia, sin límites. La IA elige defaults genéricos.
✅ TEXTO CONTROLADO

"Post LinkedIn sobre trabajo remoto. Audiencia: CTOs y engineering managers. Tono: directo, sin motivational BS. Largo: 150-180 palabras. Evitar: emojis excesivos, 'en el mundo actual'."

✅ Output predecible. Voz, estructura y límites definidos.
✍️

Control de Voz y Tono

Voz: [Técnica / Conversacional / Académica] Persona: [Experto / Colega / Mentor] Registros prohibidos: - Sin frases de relleno ("en el dinámico mundo de...") - Sin conclusiones moralizantes - Sin uso de: "robusto", "sinergias", "holístico" Fórmula de densidad: Cada párrafo = 1 idea. Si hay 2 ideas = 2 párrafos.

§ 07 — Validación

Checklist Pre-Envío

Antes de enviar cualquier prompt, verifica estos puntos. Marca cada uno mentalmente. Si fallas 3+, reescribe.

¿La instrucción principal está en las primeras 2 líneas?
Lo primero se pondera más. No entierres la tarea.
¿Especificaste el formato de salida exacto?
JSON, tabla, lista, párrafo, código... Si no dices qué, inventa.
¿Eliminaste todo saludo, preámbulo y cortesía?
"Hola", "por favor", "si puedes" = tokens que no aportan valor.
¿Incluyes al menos 1 restricción de qué NO hacer?
"Sin intro", "sin emojis", "máx X palabras". Las negaciones son eficientes.
¿Usas delimitadores para separar datos de instrucciones?
###, ---, <>, []. Previene que la IA confunda datos con órdenes.
¿Tienes solo 1 tarea por prompt?
2+ tareas = dilución. Encadena prompts, no los apiles.
¿Incluyes un ejemplo si el patrón es no-obvio?
Few-shot > explicación. 1 ejemplo bien hecho = 50 palabras ahorradas.
¿El rol asignado es específico (no genérico)?
"Copywriter direct-response SaaS" ≠ "escritor". Más específico = más enfocado.
Puntuación: 0/8
§ 08 — Errores

Errores que Queman Tokens

Los errores más comunes y cuántos tokens desperdician por cada vez que los cometes.

⚠️ "Actúa como..." sin contexto

Decir "actúa como un experto" sin definir qué expertise, en qué sector, con qué metodología = rol vacío que no filtra nada.

~40 tokens/iteración desperdiciados
⚠️ Pedir "hazlo mejor" sin especificar

La IA no sabe qué es "mejor" para ti. Di exactamente: "reduce longitud 50%", "cambia tono a formal".

~150 tokens/ciclo desperdiciados
⚠️ No limitar longitud de respuesta

Sin límite, la IA genera hasta el máximo permitido. Si necesitas 3 líneas y genera 20 párrafos = tokens quemados.

~800 tokens/exceso desperdiciados
⚠️ Pegar contexto sin delimitar

Si pegas 2000 palabras de contexto sueltas, la IA no sabe dónde empieza tu instrucción. Usa siempre delimitadores.

~300 tokens de confusión
⚠️ Usar "piensa paso a paso" sin necesidad

CoT quema 200-500 tokens extra. Úsalo SOLO para matemáticas o lógica compleja. Para texto = desperdicio.

~350 tokens innecesarios
⚠️ Repetir contexto en cada prompt

No repitas en cada prompt el contexto completo del proyecto. Usa un "system prompt" o bloque de contexto fijo.

~200 tokens/prompt desperdiciados
